# 场景分割工具 (Scene Segmentation Tool)

独立的智能场景分割工具，支持使用TransNetV2深度学习模型自动检测视频场景边界。

## 功能特性

✨ **双模式分割**
- 🤖 **智能场景分割**: 使用TransNetV2模型自动检测场景边界
- ⏱️ **普通场景分割**: 按固定/随机时间间隔分割视频

🎯 **灵活配置**
- 可设置最短/最长时间区间约束
- 三种短片段处理策略（保留/删除/合并）
- 支持GPU硬件加速（NVIDIA/AMD）
- 批量处理多个视频文件

📂 **批处理支持**
- 自动扫描文件夹中的所有视频
- 为每个视频创建独立的输出子文件夹
- 实时显示处理进度

## 系统要求

- **操作系统**: Windows 10/11
- **Python**: 3.8 或更高版本
- **FFmpeg**: 需要安装FFmpeg（用于视频处理）
- **硬盘空间**: 至少2GB可用空间（TensorFlow占用约500MB）

## 安装步骤

### 1. 安装Python依赖

```bash
pip install -r requirements.txt
```

### 2. 安装FFmpeg

**选项A: 使用Chocolatey（推荐）**
```bash
choco install ffmpeg
```

**选项B: 手动安装**
1. 从 [FFmpeg官网](https://ffmpeg.org/download.html) 下载Windows版本
2. 解压到任意位置（例如：`C:\ffmpeg`）
3. 将FFmpeg的bin目录添加到系统PATH环境变量

### 3. 验证安装

```bash
python main.py
```

## 使用方法

### GUI界面使用

1. **启动程序**
   ```bash
   python main.py
   ```

2. **配置参数**
   - 选择分割模式（智能/普通）
   - 设置时间范围（最短/最长时间）
   - 选择短片段处理策略
   - 配置硬件加速选项

3. **选择文件夹**
   - 输入文件夹：包含要分割的视频文件
   - 输出文件夹：保存分割后的视频片段

4. **开始分割**
   - 点击"开始分割"按钮
   - 等待处理完成
   - 查看分割结果

### 配置说明

#### 分割模式

- **🤖 智能场景分割**: 
  - 使用AI模型自动检测场景变化
  - 更准确，但处理速度较慢
  - 需要TensorFlow支持

- **⏱️ 普通场景分割**: 
  - 按固定或随机时间间隔分割
  - 处理速度快
  - 不需要TensorFlow

#### 时间范围配置

- **最短时间区间**: 每个片段的最小时长（秒）
- **最长时间区间**: 每个片段的最大时长（秒）
- 如果两个值相等，则生成固定时长的片段

#### 短片段处理策略

- **保留所有短片段**: 不做任何处理，输出所有片段
- **删除小于最短时长的片段**: 自动删除过短的片段
- **合并短片段**: 将短片段合并到相邻片段

#### 硬件加速

- **自动选择**: 程序自动检测最佳加速方式
- **NVIDIA GPU**: 使用CUDA加速（需要NVIDIA显卡）
- **AMD GPU**: 使用OpenCL加速（需要AMD显卡）
- **仅CPU**: 不使用硬件加速

## 故障排除

### 问题: "未找到FFmpeg"

**解决方案**:
1. 确认FFmpeg已正确安装
2. 验证FFmpeg在系统PATH中：
   ```bash
   ffmpeg -version
   ```
3. 如果不在PATH中，重新安装或手动添加到PATH

### 问题: "TensorFlow不可用"

**解决方案**:
1. 重新安装TensorFlow:
   ```bash
   pip install tensorflow-cpu>=2.10.0
   ```
2. 如果仍然失败，使用"普通场景分割"模式

### 问题: 处理速度很慢

**解决方案**:
1. 启用GPU硬件加速
2. 使用"普通场景分割"而非"智能场景分割"
3. 减小视频分辨率或时长

### 问题: 分割结果不准确

**解决方案**:
1. 调整时间范围参数
2. 尝试不同的短片段处理策略
3. 如果使用智能分割，可以尝试普通分割作为对比

## 限制与注意事项

⚠️ **重要提示**:
- 智能分割需要TensorFlow，首次运行可能需要下载模型
- GPU加速需要安装相应的驱动程序（CUDA for NVIDIA, OpenCL for AMD）
- 处理大文件时需要足够的磁盘空间
- 建议在处理前备份原始视频文件
