# 场景分割工具 - 打包说明

## 📦 打包配置

本程序使用 PyInstaller 打包成单个可执行文件 (.exe)，无需额外的文件夹或依赖。

### ✨ 主要特性

- ✅ **单文件打包** - 所有依赖和资源都打包在一个 `.exe` 文件中
- ✅ **无控制台窗口** - 程序运行时不会显示命令行窗口
- ✅ **FFmpeg 静默运行** - 视频处理时 FFmpeg 的命令行窗口也会被隐藏
- ✅ **自包含** - 包含所有必要的运行时库和 AI 模型

## 🚀 如何打包

### 方式1：使用批处理脚本（推荐）

双击运行 `build.bat` 文件，脚本会自动：

1. 检查并安装 PyInstaller
2. 清理旧的打包文件
3. 使用 `build.spec` 配置进行打包
4. 在 `dist` 文件夹中生成单个可执行文件

### 方式2：手动打包

```batch
# 1. 安装 PyInstaller（如果还没装）
pip install pyinstaller

# 2. 清理旧文件
rmdir /s /q build dist

# 3. 执行打包
pyinstaller build.spec
```

## 📁 打包后的文件

打包完成后，在 `dist` 文件夹中会生成：

```
dist/
  └── 场景分割工具.exe    （单个可执行文件，约 500MB-1GB）
```

### 为什么文件这么大？

因为打包文件包含了：
- Python 运行时环境
- TensorFlow 深度学习库
- TransNetV2 AI 模型权重
- MoviePy 视频处理库
- PyQt6 图形界面库
- 所有其他依赖库

## 🔧 配置说明

### build.spec 配置文件

关键配置项：

```python
# 隐藏控制台窗口
console=False

# 单文件模式 - 所有文件打包到 EXE 中
exe = EXE(
    pyz,
    a.scripts,
    a.binaries,    # 包含所有二进制文件
    a.zipfiles,    # 包含所有 zip 文件
    a.datas,       # 包含所有数据文件
    ...
)
```

### FFmpeg 静默运行

在代码中已经处理了 FFmpeg 的控制台窗口：

```python
# main.py 中的配置
creationflags = subprocess.CREATE_NO_WINDOW if sys.platform == 'win32' else 0
result = subprocess.run(cmd, ..., creationflags=creationflags)
```

这确保了在 Windows 上运行 FFmpeg 时不会显示命令行窗口。

## ⚠️ 注意事项

### 1. FFmpeg 依赖

打包后的程序仍然需要系统安装 FFmpeg：

- FFmpeg 必须在系统 PATH 中
- 或者将 `ffmpeg.exe` 放在与程序相同的目录下

### 2. 首次运行

- 首次运行可能需要 5-10 秒来解压临时文件
- 程序会在系统临时目录创建解压文件
- 关闭程序后临时文件会自动清理

### 3. 杀毒软件

某些杀毒软件可能会将打包的 .exe 文件标记为可疑，这是误报。可以：

- 添加到杀毒软件白名单
- 使用代码签名证书签名（专业版）

### 4. 运行环境

- 支持 Windows 7 及以上版本
- 需要 64 位系统
- 建议 8GB 以上内存（用于 AI 智能分割）

## 📊 性能优化

### UPX 压缩

配置文件中启用了 UPX 压缩来减小文件大小：

```python
upx=True,
upx_exclude=[],
```

如果打包失败，可以尝试禁用 UPX：

```python
upx=False,
```

### 排除不需要的模块

在 `build.spec` 中已经排除了不需要的大型库：

```python
excludes=[
    'matplotlib',
    'scipy',
    'pandas',
    'IPython',
    'jupyter',
    'notebook',
    'pytest',
]
```

## 🐛 常见问题

### Q: 打包后运行报错找不到 TensorFlow？

A: 确保 `build.spec` 中包含了所有 TensorFlow 数据文件和隐藏导入。

### Q: 打包后文件太大（超过 1.5GB）？

A: 可以尝试：
- 禁用 UPX 压缩可能反而会更小
- 检查是否包含了不必要的文件
- 使用 `--exclude-module` 排除更多模块

### Q: 运行时还是看到控制台窗口？

A: 确保：
- `build.spec` 中 `console=False`
- 代码中使用了 `CREATE_NO_WINDOW` 标志运行 FFmpeg

### Q: 打包时间太长？

A: 正常情况下打包需要 5-15 分钟，这是因为：
- TensorFlow 库很大
- 需要收集和打包大量依赖
- UPX 压缩需要时间
